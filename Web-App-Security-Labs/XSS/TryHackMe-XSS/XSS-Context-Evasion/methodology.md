# Testing Methodology

## Step 1 – Identify context

The injection point was analyzed to determine how input is reflected in the application.

## Step 2 – Payload adaptation

Different payloads were tested based on the context (HTML, attribute, JavaScript).

## Step 3 – Filter analysis

Input filtering mechanisms were analyzed to identify weaknesses.

## Step 4 – Evasion techniques

Payloads were modified to bypass filters and achieve execution.

## Result

The application filtering mechanisms were bypassed using context-aware payloads.
