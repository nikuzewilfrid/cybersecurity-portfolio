# Stored XSS (Persistent XSS)

## Overview

Stored XSS occurs when malicious input is stored on the server and later executed in the browser of users who view the affected content.

## Vulnerability Type

Cross-Site Scripting – Stored (Persistent)

## Key Concept

The payload is permanently stored in the application (e.g., database, comments, profiles).

Every user accessing the affected content is impacted.

## Skills Practiced

- identifying storage points (forms, comments)
- injecting persistent payloads
- understanding impact across multiple users
