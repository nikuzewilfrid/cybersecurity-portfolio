# Linux Hardening Strategy

This document contains all hardening measures applied to the audited Linux system.  
The objective is to reduce attack surface, reinforce authentication, secure network exposure, and improve logging/monitoring.

Hardening areas:

1. System Updates & Patch Management  
2. SSH Security  
3. Firewall Configuration  
4. Service Hardening  
5. File Permissions & Sensitive Files  
6. Password & Authentication Policy  
7. Kernel-Level Hardening (sysctl)  
8. Monitoring & Logging Improvements
